# Courses, Enrollment & Progress API

> Base URL: all endpoints are prefixed with `/api`

This document covers three modules your frontend will interact with most:

- **Courses** — browsing, course detail, curriculum management
- **Enrollments** — enrolling/unenrolling, listing enrolled courses
- **Progress** — lesson/course progress tracking, XP, streaks, watch time, dashboard

---

## Authentication

All endpoints (except the public course routes) require a valid session cookie set by the auth module. Include credentials on every request.

```ts
// Example fetch wrapper
const api = (path: string, init?: RequestInit) =>
  fetch(`/api${path}`, { credentials: 'include', ...init });
```

### Roles

| Role | Description |
|------|-------------|
| `STUDENT` | Enrolled learner |
| `INSTRUCTOR` | Course creator |
| `ADMIN` | Platform administrator |

## Courses Module

Base path: `/api/courses`

### Curriculum visibility rules

| Caller | Endpoint | Curriculum depth |
|--------|----------|-----------------|
| Anyone (no auth) | `GET /courses/public` or `GET /courses/public/:id` | Outline only (title + position) |
| STUDENT, not enrolled | `GET /courses/:id` | Outline only + `isEnrolled: false` |
| STUDENT, enrolled | `GET /courses/:id` | Full (duration, isPreview, materials) + `isEnrolled: true` |
| INSTRUCTOR (own course) | `GET /courses/:id` | Full |
| ADMIN | `GET /courses/:id` | Full |

### GET /courses/public

Browse published courses. No authentication required. Always returns `PUBLISHED` courses only.

**Query parameters**

| Param | Type | Description |
|-------|------|-------------|
| `search` | `string` | Full-text search across title, description, and tags |
| `level` | `BEGINNER \| INTERMEDIATE \| ADVANCED` | Filter by difficulty |
| `certification` | `NORMAL \| PROFESSIONAL \| ADVANCED` | Filter by certification type |
| `tags` | `string[]` | Course must contain ALL specified tags |
| `minPrice` | `number` | Minimum price |
| `maxPrice` | `number` | Maximum price |
| `cursor` | `string` | Last seen course `id` — omit for first page |
| `limit` | `number` | Page size (default `20`) |
| `sortBy` | `createdAt \| price \| totalRating \| publishedAt \| title` | Sort field (default `createdAt`) |
| `sortOrder` | `asc \| desc` | Sort direction (default `desc`) |

**Response** `200`

```ts
interface PaginatedCourses {
  items: CourseListItem[];   // Course + sections (outline only) + totalLectures
  nextCursor: string | null; // null = last page
}
```

**Example**

```ts
const res = await fetch('/api/courses/public?search=nestjs&limit=10');
const { items, nextCursor } = await res.json();
```

## Progress Module

Base path: `/api/progress`

All progress endpoints require `STUDENT` role unless noted otherwise.

### How progress works

1. The video player sends periodic **pings** via `PATCH /progress/lessons/:lessonId` with the current playback position in seconds.
2. The server stores a **high-water mark** — `watchedSeconds` never decreases.
3. A lesson is **auto-completed** when `watchedSeconds / duration >= 0.8` (80%).
4. Students can also manually complete a lesson via `POST /progress/lessons/:lessonId/complete`.
5. On each completion, the server recomputes `CourseProgress`, updates the **learning streak**, and awards **XP**.

### Sequential lesson locking

Lessons are locked until the previous lesson has been watched at least **90%** or is marked complete. Preview lessons (`isPreview: true`) and the very first lesson of a course are always unlocked.

The `isLocked` field is returned on every lesson in progress detail responses.

### GET /progress/xp

Student XP and level state. Role: `STUDENT`.

Each completed lesson awards **45 XP**. Level is capped at **9** but XP keeps accumulating.

**XP thresholds**

| Level | Cumulative XP required |
|-------|----------------------|
| 0 | 0 |
| 1 | 450 |
| 2 | 700 |
| 3 | 1,000 |
| 4 | 1,200 |
| 5 | 1,500 |
| 6 | 1,900 |
| 7 | 2,400 |
| 8 | 3,000 |
| 9 (max) | 3,750 |

**Response** `200`

```ts
interface XPState {
  totalXp: number;
  currentLevel: number;          // 0–9
  xpToNextLevel: number;         // 0 when at max level
  xpIntoCurrentLevel: number;    // XP accumulated inside current level bracket
  currentLevelSpan: number;      // total XP span of current level bracket (0 at max)
  maxLevel: number;              // always 9
  recentAwards: Array<{
    lessonId: string;
    xpAwarded: number;
    awardedAt: string;
  }>;
}
```

**Example — rendering a level progress bar**

```ts
const { xpIntoCurrentLevel, currentLevelSpan, currentLevel, xpToNextLevel } = xpState;

// percentage filled in the current level bracket
const pct = currentLevelSpan > 0
  ? (xpIntoCurrentLevel / currentLevelSpan) * 100
  : 100; // at max level
```

### PATCH /progress/lessons/:lessonId

Video player progress ping. Role: `STUDENT`.

Call this periodically (e.g. every 10–30 seconds) while the student is watching a lesson. The server stores a high-water mark — sending a lower value than previously recorded is a no-op.

**Body**

```ts
interface UpdateLessonProgressDto {
  watchedSeconds: number;  // current playback position in seconds (>= 0)
}
```

**Response** `200`

```ts
interface LessonProgressResponse {
  lessonId: string;
  watchedSeconds: number;
  duration: number;
  percentWatched: number;
  isCompleted: boolean;
  completedAt: string | null;
  lastWatchedAt: string;
  xpAwarded?: number;  // only present when the lesson was just completed by this ping
}
```

**Errors**

| Status | Reason |
|--------|--------|
| `403` | Not enrolled in this course |
| `403` | Lesson is locked (previous lesson not yet 90% watched) |
| `404` | Lesson not found |

**Example — video player integration**

```ts
let pingTimer: ReturnType<typeof setInterval>;

function startProgressPing(lessonId: string) {
  pingTimer = setInterval(async () => {
    const currentTime = videoPlayer.currentTime; // seconds
    const res = await api(`/progress/lessons/${lessonId}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ watchedSeconds: Math.floor(currentTime) }),
    });
    const data = await res.json();
    if (data.xpAwarded) {
      showXPToast(data.xpAwarded); // e.g. "+45 XP"
    }
  }, 15_000); // ping every 15 seconds
}

function stopProgressPing() {
  clearInterval(pingTimer);
}
```

## Common Patterns

### Cursor-based pagination

All list endpoints use cursor pagination. Pass the `nextCursor` from the previous response as the `cursor` query param to get the next page. A `null` `nextCursor` means you're on the last page.

```ts
async function fetchAllCourses() {
  const courses = [];
  let cursor: string | null = undefined;

  do {
    const params = new URLSearchParams({ limit: '50' });
    if (cursor) params.set('cursor', cursor);

    const res = await api(`/courses/public?${params}`);
    const { items, nextCursor } = await res.json();

    courses.push(...items);
    cursor = nextCursor;
  } while (cursor !== null);

  return courses;
}
```

## Storage Module

Base path: `/api/storage`

The storage module handles all file uploads for the platform using **Cloudflare R2** (S3-compatible). The NestJS server never handles raw file bytes — instead it issues short-lived pre-signed URLs and the client uploads directly to R2.

### Upload flow

```
Client                          NestJS API                    Cloudflare R2
  │                                  │                               │
  │  POST /api/storage/presigned-upload                              │
  │  { folder, fileName, contentType, contentLength }                │
  │ ────────────────────────────────>│                               │
  │                                  │  validates type + size        │
  │                                  │  generates signed PUT URL     │
  │  { uploadUrl, key, publicUrl }   │                               │
  │ <────────────────────────────────│                               │
  │                                  │                               │
  │  PUT {uploadUrl}  (raw file bytes, Content-Type header)          │
  │ ─────────────────────────────────────────────────────────────── >│
  │                                  │                               │
  │  204 No Content                  │                               │
  │ <─────────────────────────────────────────────────────────────── │
  │                                  │                               │
  │  Store key / publicUrl in your resource (course, lesson, etc.)   │
```

### Folder reference

| Folder | `folder` value | Allowed MIME types | Max size |
|--------|---------------|-------------------|----------|
| Course thumbnails | `course-images` | `image/jpeg`, `image/png`, `image/webp` | 5 MB |
| Course intro videos | `course-videos` | `video/mp4`, `video/webm`, `video/quicktime` | 2 GB |
| Lesson materials | `lesson-materials` | PDF, audio (mp3/ogg/wav), video (mp4/webm/quicktime), images | 500 MB |
| User avatars | `avatars` | `image/jpeg`, `image/png`, `image/webp`, `image/gif` | 2 MB |
| Assignment files | `assignments` | PDF, `video/mp4`, images, `application/zip` | 50 MB |
| Course badges | `course-badges` | `image/jpeg`, `image/png`, `image/webp`, `image/svg+xml` | 1 MB |
| Chat attachments | `chat-attachments` | Images, PDF, video, audio, zip, Word, Excel, plain text | 25 MB |

### POST /storage/presigned-upload

Get a pre-signed PUT URL for a direct-to-R2 upload. Roles: `STUDENT`, `INSTRUCTOR`, `ADMIN`.

**Body**

```ts
interface PresignedUploadRequestDto {
  folder: StorageFolder;    // see folder reference above
  fileName: string;         // original file name — used to derive the extension
  contentType: string;      // MIME type, e.g. "video/mp4"
  contentLength: number;    // file size in bytes
}
```

**Response** `201`

```ts
interface PresignedUploadResponse {
  uploadUrl: string;   // pre-signed PUT URL — expires in `expiresIn` seconds
  key: string;         // R2 object key — store this as publicId / key in your DB record
  publicUrl: string;   // CDN public URL — available after the PUT completes
  expiresIn: number;   // seconds until uploadUrl expires (default 900 = 15 min)
}
```

**Errors**

| Status | Reason |
|--------|--------|
| `400` | MIME type not allowed in the target folder |
| `400` | File size exceeds the folder limit |
| `401` | Unauthenticated |

---

### GET /storage/signed-url?key=

Get a temporary signed GET URL for a **private** R2 object. All authenticated roles.

Use this for private assets like assignment submissions. Public assets (course thumbnails, intro videos) should be served directly via their `publicUrl` — no signed URL needed.

**Query parameter**

| Param | Description |
|-------|-------------|
| `key` | The R2 object key returned by the presigned-upload endpoint |

**Response** `200`

```ts
interface SignedViewUrlResponse {
  url: string;        // temporary signed GET URL
  expiresIn: number;  // seconds until URL expires (default 3600 = 1 hour)
}
```

**Errors**

| Status | Reason |
|--------|--------|
| `401` | Unauthenticated |
| `404` | No object exists at the given key |

### Course-related upload workflows

#### Uploading a course thumbnail (`img`)

```ts
// 1. Get a pre-signed URL
const { uploadUrl, key, publicUrl } = await api('/storage/presigned-upload', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    folder: 'course-images',
    fileName: file.name,
    contentType: file.type,
    contentLength: file.size,
  }),
}).then(r => r.json());

// 2. Upload directly to R2
await fetch(uploadUrl, {
  method: 'PUT',
  headers: { 'Content-Type': file.type },
  body: file,
});

// 3. Save the publicUrl when creating/updating the course
await api('/courses', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    title: 'My Course',
    price: 49.99,
    img: publicUrl,   // ← use publicUrl here
    // ...
  }),
});
```

#### Uploading a course intro video (`videoUrl`)

Same flow as above but with `folder: 'course-videos'`. The `publicUrl` goes into the `videoUrl` field on the course.

```ts
const { uploadUrl, publicUrl } = await api('/storage/presigned-upload', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    folder: 'course-videos',
    fileName: videoFile.name,
    contentType: videoFile.type,   // e.g. "video/mp4"
    contentLength: videoFile.size,
  }),
}).then(r => r.json());

await fetch(uploadUrl, {
  method: 'PUT',
  headers: { 'Content-Type': videoFile.type },
  body: videoFile,
});

// Use publicUrl as the videoUrl when creating/updating the course
```

> Large video uploads (up to 2 GB) can take time. Show a progress bar using the `XMLHttpRequest` `upload.onprogress` event or the `fetch` streaming API.

#### Uploading a lesson material

```ts
const { uploadUrl, key, publicUrl } = await api('/storage/presigned-upload', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    folder: 'lesson-materials',
    fileName: file.name,
    contentType: file.type,   // e.g. "application/pdf"
    contentLength: file.size,
  }),
}).then(r => r.json());

await fetch(uploadUrl, {
  method: 'PUT',
  headers: { 'Content-Type': file.type },
  body: file,
});

// Attach the material to the lesson
await api(`/courses/${courseId}/sections/${sectionId}/lessons/${lessonId}/materials`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    type: 'PDF',           // VIDEO | PDF | AUDIO | LINK
    title: 'Lesson Slides',
    url: publicUrl,        // ← CDN public URL
    publicId: key,         // ← store the key for future deletion
    fileName: file.name,
    size: file.size,
  }),
});
```

> Always store the `key` as `publicId` on the material record. This is what you pass to `DELETE /storage/file` when the material is removed.

#### Deleting a material and its file

When the instructor removes a lesson material, delete the R2 object too:

```ts
// 1. Remove the material record
await api(`/courses/${courseId}/sections/${sectionId}/lessons/${lessonId}/materials/${materialId}`, {
  method: 'DELETE',
});

// 2. Delete the file from R2 (use the publicId you stored)
await api('/storage/file', {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ key: material.publicId }),
});
```

### Key vs publicUrl — when to use which

| Field | When to use |
|-------|-------------|
| `publicUrl` | Store in the DB and render in the UI for **public** assets (course images, videos, lesson materials). Serve directly — no extra API call needed. |
| `key` | Store as `publicId` in the DB. Use it to call `DELETE /storage/file` when the asset is removed. Also pass to `GET /storage/signed-url?key=` for **private** assets. |

Public assets (everything in `course-images`, `course-videos`, `lesson-materials`, `course-badges`) are served via the CDN public domain and do not need a signed view URL. Only use `GET /storage/signed-url` for private assets like assignment submissions.
